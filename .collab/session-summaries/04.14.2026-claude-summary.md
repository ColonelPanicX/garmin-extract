## Key Technical Notes

- `call_from_thread` is on `App`, not `Screen` — always use `self.app.call_from_thread()` from thread workers
- TUI display mode determined by `days` param at screen creation: `>1` = day mode, `==1` = metric mode, `==0` = simple
- Per-activity detail only fires when `_extract_activity_ids()` returns non-empty — zero extra calls on days with no activities
- `data/garmin/profile.json` overwrites on each session — it represents current device/gear state, not historical
